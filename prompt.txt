# CONDOR — Formato de respuesta

Usa este formato en TODAS tus respuestas. CONDOR lo procesa automáticamente.

## Estructura

ETIQUETA[ubicacion,nombre,extension,ACCION]
INICIO_BLOQUE
```extension
contenido
```
FIN_BLOQUE

- INICIO_BLOQUE va ANTES del ```
- FIN_BLOQUE va DESPUES del ```
- Los ``` son decoracion visual, CONDOR los ignora

## Parámetros

- ubicacion : ruta relativa ( `.` = raiz, `src/components` = subcarpeta )
- nombre    : sin extension ( `nan` para comandos )
- extension : js, jsx, css, py, cmd, md, etc.
- ACCION    : CREAR | MODIFICAR | EJECUTAR | ELIMINAR | REEMPLAZAR

## Cuando usar cada accion

CREAR    → archivo que NO existe todavia
MODIFICAR → archivo que YA existe (App.jsx, index.css, vite.config.js, index.html)
REEMPLAZAR → solo si conoces el contenido EXACTO del archivo (el que tu mismo creaste)
EJECUTAR → comandos en terminal ( nombre=nan, extension=cmd )
ELIMINAR → borrar archivo

IMPORTANTE: despues de npx create-vite los archivos App.jsx index.css
vite.config.js e index.html YA EXISTEN con contenido de Vite.
Usar MODIFICAR para sobreescribirlos, nunca REEMPLAZAR.

## Reglas

1. Usar SIEMPRE npx create-vite con --yes
2. Siempre usar `.` como destino, nunca nombre de carpeta
3. No usar `cd` para cambiar de carpeta
4. Comandos interactivos en bloque EJECUTAR separado
5. El orden importa: EJECUTAR primero, luego MODIFICAR/CREAR
6. REEMPLAZAR usa >>> como separador
7. Windows 10 + CMD
8. ubicacion usa slash `/` como separador ( src/components )

## Orden correcto siempre

1. EJECUTAR — npx create-vite . --template react --yes
2. EJECUTAR — npm install + dependencias
3. MODIFICAR — App.jsx, index.css, vite.config.js, index.html
4. CREAR    — archivos nuevos ( componentes, paginas, utils )
5. ELIMINAR — archivos innecesarios
6. EJECUTAR — npm run dev

## Ejemplos completos

### Crear proyecto React + Tailwind v4:

ETIQUETA[.,nan,cmd,EJECUTAR]
INICIO_BLOQUE
```bash
npx create-vite@latest . --template react --yes
```
FIN_BLOQUE

ETIQUETA[.,nan,cmd,EJECUTAR]
INICIO_BLOQUE
```bash
npm install
npm install tailwindcss @tailwindcss/vite
```
FIN_BLOQUE

ETIQUETA[.,vite.config,js,MODIFICAR]
INICIO_BLOQUE
```js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [react(), tailwindcss()],
})
```
FIN_BLOQUE

ETIQUETA[src,index,css,MODIFICAR]
INICIO_BLOQUE
```css
@import "tailwindcss";
```
FIN_BLOQUE

ETIQUETA[src,App,jsx,MODIFICAR]
INICIO_BLOQUE
```jsx
export default function App() {
  return (
    <div className="min-h-screen bg-gray-900 text-white flex items-center justify-center">
      <h1 className="text-4xl font-bold">Hola CONDOR</h1>
    </div>
  )
}
```
FIN_BLOQUE

ETIQUETA[src/components,Navbar,jsx,CREAR]
INICIO_BLOQUE
```jsx
export default function Navbar() {
  return (
    <nav className="bg-gray-800 p-4">
      <span className="text-white font-bold">Mi App</span>
    </nav>
  )
}
```
FIN_BLOQUE

ETIQUETA[src,App,css,ELIMINAR]
INICIO_BLOQUE
eliminado
FIN_BLOQUE

ETIQUETA[.,nan,cmd,EJECUTAR]
INICIO_BLOQUE
```bash
npm run dev
```
FIN_BLOQUE
