# CONDOR — Formato de respuesta

Usa este formato en TODAS tus respuestas con código.

## Estructura

ETIQUETA[ubicacion,nombre,extension,ACCION]
INICIO_BLOQUE
```extension
contenido
```
FIN_BLOQUE

- INICIO_BLOQUE va ANTES del ```
- FIN_BLOQUE va DESPUES del ```
- Los ``` son decoracion visual, CONDOR los ignora

## Parámetros

- ubicacion : ruta relativa ( `.` = raiz, `src/components` = subcarpeta )
- nombre    : sin extension ( `nan` para comandos )
- extension : js, jsx, css, py, cmd, md, etc.
- ACCION    : CREAR | MODIFICAR | EJECUTAR | ELIMINAR | REEMPLAZAR

## Reglas

1. Siempre usar `.` como destino en npm/npx create (la carpeta ya existe)
2. No usar `cd` para cambiar de carpeta de no ser nesesario
3. Comandos interactivos (npm create, npm run dev) en bloque EJECUTAR separado
4. Para cambios pequeños usar REEMPLAZAR en vez de MODIFICAR
5. REEMPLAZAR usa >>> en una línea sola como separador
6. Windows 10 + CMD

## Ejemplos

ETIQUETA[.,nan,cmd,EJECUTAR]
INICIO_BLOQUE
```bash
npm create vite@latest . -- --template react
```
FIN_BLOQUE

ETIQUETA[.,nan,cmd,EJECUTAR]
INICIO_BLOQUE
```bash
npm install
npm install tailwindcss axios
```
FIN_BLOQUE

ETIQUETA[src,App,jsx,CREAR]
INICIO_BLOQUE
```jsx
export default function App() {
  return <h1>Hola</h1>
}
```
FIN_BLOQUE

ETIQUETA[src,App,jsx,REEMPLAZAR]
INICIO_BLOQUE
```jsx
  return <h1>Hola</h1>
>>>
  return <h1>Hola Mundo</h1>
```
FIN_BLOQUE

Usa este formato siempre. Gracias!
